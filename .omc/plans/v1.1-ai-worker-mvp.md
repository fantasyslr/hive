# Hive v1.1: AI Worker MVP

## Core Goal

用户打开看板，看到 AI 在自动干活。任务从 Pending → Claimed → Working → Done，中间没有人碰过。

## Why This Matters

v1 做了 3 个 Phase 全是基础设施。Hive 和 Trello 的唯一区别是"AI 做重活"——这个还没实现。v1.1 的唯一目标是让这个画面出现。

## Requirements Summary

| ID | Requirement | Phase |
|----|-------------|-------|
| WK-01 | Worker bridge 能注册为 agent、领任务、自动执行、回写结果 | 1 |
| WK-02 | Worker bridge 兼容 v1 camelCase API + Bearer auth | 1 |
| WK-03 | Worker adapter 对接至少一个真实 LLM CLI (Claude Code) | 1 |
| WK-04 | Worker 执行前自动从 memory 拉取相关上下文注入 prompt | 1 |
| UI-01 | Agent Bar 实时显示 AI worker 在线状态 | 2 |
| UI-02 | 任务卡片显示 AI worker 执行进度（Working 状态 + assignee） | 2 |
| UI-03 | TaskDetail 展示 AI 产出的 result 内容 | 2 |
| UI-04 | SSE 实时推送 AI worker 状态变化（无需刷新） | 2 |
| E2E-01 | 端到端演示：创建任务 → AI 自动领取 → 执行 → 结果显示在看板 | 3 |

## Phases

### Phase 1: Worker Bridge 修复 + LLM 接入（后端）

**Goal**: `npm run worker:claude` 启动后，Claude CLI 作为 agent 自动注册、领任务、执行、回写结果。

**Tasks**:

1. **修复 worker-bridge.ts 兼容 v1 API**
   - `scripts/worker-bridge.ts:84` — `agent_id` → `agentId`（注册请求）
   - `scripts/worker-bridge.ts:85-89` — 注册 payload 全部改 camelCase
   - `scripts/worker-bridge.ts:66-79` — `http()` 函数加 `Authorization: Bearer <token>` header（从 `HIVE_TOKEN` 环境变量读取）
   - `scripts/worker-bridge.ts:201,212,219` — PATCH payload `agent_id` → `agentId`
   - 写测试验证修复

2. **创建 worker-adapter.sh**
   - `scripts/worker-adapter.sh` — 读取 `HIVE_TASK_JSON` 环境变量，构造 prompt，调用对应 CLI
   - Claude adapter: `claude -p "你是 Hive AI worker。任务：${TITLE}\n描述：${DESC}\n请完成这个任务并输出结果。"`
   - Gemini adapter: `gemini -p ...` (结构类似)
   - 输出写到 stdout，bridge 自动捕获

3. **Memory 上下文注入**
   - 在 `processClaimedTask()` 中，执行前调用 `GET /memory/search?query={task.title}&limit=3`
   - 把 memory 结果拼入 worker prompt："以下是相关历史记录：..."
   - 让 AI worker 站在团队积累的知识上工作，而不是从零开始

4. **端到端 smoke test**
   - `scripts/smoke-test-worker.sh` — 启动 gateway + 启动 claude worker + 创建任务 + 等待 done + 验证 result 非空

**Files changed**: `scripts/worker-bridge.ts`, `scripts/worker-adapter.sh` (new), `scripts/smoke-test-worker.sh` (new)

**Acceptance Criteria**:
- [ ] `npm run worker:claude` 启动后，agent 出现在 `GET /board` 的 agents 列表
- [ ] 创建一个 task with `requiredCapabilities: ['planning']`，claude worker 自动 claim → working → done
- [ ] task.result 包含 Claude 生成的真实内容（非空、非 mock）
- [ ] task.result 中引用了 memory 上下文（如果有的话）
- [ ] 全程无手动干预

### Phase 2: 前端 AI 体验（前端）

**Goal**: 用户在浏览器里看到 AI worker 在线、正在干活、产出了什么。

**Tasks**:

1. **Agent Bar 增强**
   - `packages/hive-ui/src/components/AgentBar.tsx` — 显示 agent 在线状态（绿点=online、灰点=offline）
   - 显示 agent name 和当前正在做的任务（如果有）
   - SSE `agent.online`/`agent.offline` 事件驱动状态变化

2. **任务卡片 Working 状态**
   - `packages/hive-ui/src/components/TaskCard.tsx` — Working 状态时显示 assignee（AI agent 名字）+ 动画/脉冲指示器
   - 让用户一眼看到"这个任务 AI 正在做"

3. **TaskDetail 展示 AI 结果**
   - `packages/hive-ui/src/components/TaskDetail.tsx` — Done 状态时展示 `task.result` 内容
   - 用 markdown 渲染（AI 产出通常是 markdown 格式）
   - Failed 状态时展示 `task.error`

4. **SSE 驱动全流程动画**
   - task.assigned → 卡片从 Pending 移到 Claimed
   - task.updated (status=working) → 卡片移到 Working 列 + 脉冲动画
   - task.completed → 卡片移到 Done 列 + result 可查看
   - task.failed → 卡片移到 Failed 列 + error 可查看

**Files changed**: `AgentBar.tsx`, `TaskCard.tsx`, `TaskDetail.tsx`, `App.tsx`（SSE event handler 增强）

**Acceptance Criteria**:
- [ ] AI worker 启动后，Agent Bar 立刻显示 agent 上线（绿点 + 名字）
- [ ] AI worker 领任务后，Pending 列卡片自动移到 Claimed → Working（无刷新）
- [ ] Working 状态卡片有视觉指示器（脉冲/动画/图标）
- [ ] AI 完成后，卡片移到 Done 列，点击详情能看到 result 内容
- [ ] AI 失败后，卡片移到 Failed 列，点击详情能看到 error

### Phase 3: 端到端演示（联合）

**Goal**: 录一个 30 秒 demo — 创建 Campaign → AI 自动拆分子任务 → 各 AI worker 分别执行 → 结果汇总到看板。

**Tasks**:
1. 同时启动 claude + gemini worker（或两个 claude worker 不同角色）
2. 从 UI 创建 Campaign 模板任务
3. 验证子任务自动分配给不同 AI worker
4. 验证所有子任务自动执行完成
5. 截图/录屏作为 demo 素材

**Acceptance Criteria**:
- [ ] 从创建 Campaign 到所有子任务 Done，全程无人干预
- [ ] 每个子任务的 result 是真实 AI 产出（非 mock）
- [ ] 用户能在看板上实时看到整个过程

## Risks and Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| Claude CLI 本地未安装 | Worker 无法执行 | 检测 CLI 可用性，fallback 到 echo mock |
| LLM API 调用耗时长(30s+) | 用户觉得卡住 | TaskCard 显示 Working 脉冲 + 开始时间 |
| Worker bridge 注册后被 auth 拒绝 | Agent 无法工作 | 启动时验证 token 有效性，失败快速报错 |
| Memory search 无结果 | Prompt 没有上下文 | 优雅降级：无 memory 时跳过注入 |

## Verification Steps

1. 跑 `npm test` 确认无回归（172 tests 全绿）
2. 启动 Gateway + Worker + UI，浏览器观察完整流程
3. 创建任务 → 30s 内看到 AI 产出结果
4. Agent 掉线 → Agent Bar 状态更新为 offline
