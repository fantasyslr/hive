**In packages/hive-gw/src/routes/tasks.ts:**

Create a helper function `filterTasksByRole` at the top of the file (after imports):

```typescript
function filterTasksByRole(tasks: Task[], user: HiveUser): Task[] {
  if (user.role === 'manager') return tasks;
  return tasks.filter(t =>
    t.createdBy === user.id ||
    t.assignee === user.id ||
    (t.assignee === null && t.status === 'pending')
  );
}
```

Import `HiveUser` type from `../config.js` (or wherever Plan 02 exported it — check the SUMMARY). Import the `Task` type if not already imported.

Update the `GET /` handler:
```typescript
tasksRouter.get('/', (req, res) => {
  let tasks = taskMachine.getAll();
  // Role-based visibility (AUTH-03)
  tasks = filterTasksByRole(tasks, req.user!);
  // Status filter
  const statusFilter = req.query.status as string | undefined;
  if (statusFilter) {
    tasks = tasks.filter(t => t.status === statusFilter as TaskStatus);
  }
  res.json(tasks);
});
```

The `GET /:id` endpoint does NOT need filtering — if a user knows the task ID, they can access it directly (single-task access is fine; the visibility rule is about list views).
