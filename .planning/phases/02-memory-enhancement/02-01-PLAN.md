    - Test: add({namespace:"ns", content:"alpha beta"}) twice → only 1 entry exists, second call returns updated record with same id
    - Test: add({namespace:"ns", content:"alpha beta"}) then add({namespace:"ns", content:"alpha beta gamma"}) with high similarity → updates existing (similarity > 0.85 threshold)
    - Test: add({namespace:"ns1", content:"alpha"}) then add({namespace:"ns2", content:"alpha"}) → 2 separate entries (different namespace = no dedup)
    - Test: add({namespace:"ns", content:"completely different topic"}) after existing entry → creates new entry (low similarity)
    - Test: dedup returns the updated record with new updatedAt timestamp
  </behavior>
  <action>
    1. In store.ts, add a private method `findDuplicate(namespace: string, content: string): MemoryRow | null`:
       - Query: SELECT * FROM memories WHERE namespace = ? AND (expires_at IS NULL OR expires_at > datetime('now'))
       - For each candidate, compute embedding similarity between new content and candidate
       - If any candidate scores > 0.85 (cosine similarity of embeddings), return it as duplicate
       - Return null if no duplicate found

    2. Modify add() to call findDuplicate() first:
       - If duplicate found: call update() on that record with the new content (merge metadata), return updated record
       - If no duplicate: proceed with INSERT as before
       - This makes add() idempotent for similar content in the same namespace

    3. Add tests for all behaviors above.
