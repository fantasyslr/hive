GET /templates — returns `getAllTemplates()`

GET /templates/:id — returns `getTemplate(id)` or 404

POST /templates/:id/launch — the core route:
  a. Get template by id (404 if not found)
  b. Accept optional `{ name?: string, description?: string }` from req.body for the parent task name override (e.g., "Q3 Japan Campaign")
  c. Create parent task via taskMachine.create():
     - title: `req.body.name || template.name`
     - description: `req.body.description || template.description || ""`
     - requiredCapabilities: ["orchestration"]
     - createdBy: `req.user!.id`
     - taskKind: "plan"
  d. Build a title-to-taskId map: `Map<string, string>`
  e. For each template task, create sub-task via taskMachine.create():
     - title: templateTask.title
     - description: templateTask.description || ""
     - requiredCapabilities: templateTask.capabilities
     - createdBy: req.user!.id
     - parentTaskId: parent.id
     - dependsOn: templateTask.dependsOn.map(title => titleToTaskId.get(title)) — resolve titles to IDs
     - Store mapping: titleToTaskId.set(templateTask.title, subTask.id)
  f. For sub-tasks with empty dependsOn, try dispatcher.autoAssign() and emit task.assigned event if successful
  g. Emit a task.updated event for the parent task with `{ templateId: template.id, subTaskCount: template.tasks.length }`
  h. Return 201 with `{ parent, subTasks, templateId: template.id }`
