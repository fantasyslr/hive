       - Checks `parentTaskId` exists via `taskMachine.get(parentTaskId)` — 400 if not found
       - Iterates `tasks` array, creates each via `taskMachine.create()` with `parentTaskId`, `runId` (use parent's runId or parentTaskId as fallback), `createdBy: 'coordinator'`
       - Builds `titleToId` map: `Map<string, string>` — after creating each task, maps its title to its ID
       - Second pass: resolves `dependsOn` title strings to IDs using titleToId map. If any title not found, rollback (delete created tasks) and return 400
       - For resolution: use `taskMachine.get(taskId)` then update the task's `dependsOn` field. Since TaskMachine stores by reference-like Map, we need a method. Since task-machine exposes `restore()` for recovery, the cleanest approach is to create tasks in two passes:

         - Pass 2: For each task with dependsOn titles, resolve to IDs, then re-set the dependsOn. Add a `setDependsOn(taskId: string, deps: string[]): Task` method to TaskMachine that updates dependsOn field.
       - Auto-assign independent tasks (those with no dependsOn or empty dependsOn) via `dispatcher.autoAssign(task)`, emit `task.assigned` events

       ```typescript
       setDependsOn(taskId: string, deps: string[]): Task | undefined {
         const task = this.tasks.get(taskId);
         if (!task) return undefined;
         const updated = { ...task, dependsOn: deps, updatedAt: new Date().toISOString() };
         this.tasks.set(taskId, updated);
         return updated;
       }
       ```
